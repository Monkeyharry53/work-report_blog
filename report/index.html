As mentioned above during my co-op, I was given the task of building an integration with PSD CityWide to sync asset data between CityWide and the Township GIS system.I first met with the client and my colleagues a few times. However, the client worked in a separate department at Township of Central Wellington. My first week's main goal was to properly understand the project, which I did by reading through a few documents and attending meetings about it.
After a few brainstorming sessions, I suggest creating a website that will facilitate communication between API and MSSQL. Additionally, the website's graphical user interface will make it much simpler for clients to do tasks using the website. Then I decided to use Python Flask as the backend language because it has numerous built-in packages that make code easy to read and understand. I tried to make my code more readable and made a separate documentation along with in code comments, keeping in mind that my code will be utilised even after my co-op.
During this project, I was able to advance my knowledge of HTML, CSS, JavaScipt, and Python Flask, among other things.I finished the stromwind course on constructing security using python flask libraries and used sqlite to store login credentials. I learnt how to develop login security using flask login and flask schemalchemy. It was my first time building security for a website, and it was a great experience.
In order to understand how to create views from many tables and apply conditions to them, I also took a T-SQL course from Stromwind. This work term assisted me in learning new concepts and gave me a better understanding of how a project is produced.
The most difficult and exciting element of the project was my final task, which was to host my website on an Apache server. I started out by watching a few YouTube videos about hosting web servers. Because I had never hosted a web server during my academic career, this was absolutely foreign to me. The server's construction and creation of the final documentation took me close to three weeks. Just learning and hosting a web server with simple HTML and CSS files took up the first week. When hosting an Apache web server, I ran into a number of issues. The first important one was a compatibility issue with several Python libraries. In order to fix this, I defined the version of the Numpy library using a pipfile. I also encountered a number of minor problems that were resolved via forms and standard debugging. The last issue was connecting my Apache web server to MSSQL, which required installing SQL command line operators and changing predefined files in Apache server.

CONCLUSION

In summary, this work term proved to be a fruitful summer as I was able to learn a lot from this experience, and improve on both my personal and technical skills. I have found a new interest in front-end development and would recommend anyone starting out in the co-op program to try their hand at it. I got the opportunity to learn from industry professionals, see how things are run within a small development team, and practice what I learned from university. I learned my strengths, and weaknesses, and have understood what areas I need to improve on in future co-op terms.
